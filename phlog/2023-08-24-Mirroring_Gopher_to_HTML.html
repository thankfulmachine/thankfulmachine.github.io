Mirroring Gopher to HTML

I'd like to mirror this content on a static HTTP server, but have been unsure up to now how to do that to serve my needs without too much complication.

On the one hand, I like writing HTML by hand, enjoy some of its basic semantic features, and basic styling like choosing colors. However, I also prefer the plain text simplicity of Gopher. I've gone back and forth on a few options:

  1. A custom HTML -> Gopher converter, writing HTML first.
  2. Lynx's --dump mode. I was surprised at the high quality of the dumps that Lynx produced.
  3. A custom Markdown -> Gopher converter.
  4. Somehow get Markdown to work as a gopher map.

What I decided was something like option 4.

I like to use the phd server, which I have forked with a few modifications for my own use at: https://github.com/thankfulmachine/phd.

This server uses the Geomyidae format for links in gopher maps, which interprets lines that look like this:

[1|My Directory of Things|/mythings]

as a link in gophermaps. Aside from those, I'm basically writing Markdown everywhere else in my plain text files and gopher maps out of habit, so I've created a very small script that replaces these link lines as:

  • A little HTML region with an icon file, a la Mosaic directory listings, or
  • An <img> tag for image links

What remains is something that cmark can convert! It creates extremely plain and boring HTML, which is exactly what I would write anyway.

cmark --unsafe -t html

I'm pretty happy with this compromise: it allows me to write the content as plain text first for my Gopher server and allow the website to fill the role of a "pretty proxy", without having to use an actual proxy server. That works for me!

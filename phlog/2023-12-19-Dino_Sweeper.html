Dino Sweeper

During my holiday break, I'm taking the time to learn some retro Windows programming. I never really took the time to learn it over the years, focusing mainly on *nix style operating systems, so I feel I missed out on a lot of the fun during the peak OS era of the late 90s and early 2000s. So, I thought a good little project would be to reimplement Mine Sweeper. It holds a special place in my heart and seems simple enough to implement.

I'm calling my little winmine clone "Dino Sweeper". Instead of searching for mines, you're searching for dinosaur fossils. Dig in the wrong spot and you've ruined a precious specimen!

I've been putting together a Pentium III machine that will run Windows 98 Second Edition and Windows 2000, where I'll be doing my development. Until I get a few remaining components, I'll work in a VM.

A couple of impressions as I revisit this universe for the first time in a couple of decades:

First, Windows 2000 is perhaps still the greatest mainstream operating system of all time, at least regarding the everyday user experience. The surface area of the OS that you need to know to live is small, the look and feel of the UI is so simple as to be downright calming, and it's fast as hell. If you're reading this phlog, I doubt I need to tell you how exhausted I must be of "modern" UIs with their endless notifications, mental noise-inducing compositing effects, and an ironic amount of emptiness.

Second, I'm using Visual Studio 6.0, released in 1998. Again, the simplicity. It launches pretty much instantaneously and everything works as I expect so far.

As I dip my toes in the Win32 API, it's essentially the same as everything else. Creating windows and menus, handling UI events, open and closing files, loading resources, etc. etc.

Nothing new under the sun.
